事件

发表于 2018-01-13 | 分类于 《JavaScript高级程序设计》读书笔记

事件流

事件流描述的是从页面中接收事件的顺序，但IE的事件流是事件冒泡流，而Netscape Communicator的事件流是事件捕获流；

1.事件冒泡

事件冒泡：事件开始时由最具体的元素（文档中嵌套层次最深的节点）接收，然后逐级向上传播到较为不具体的节点（文档）；
IE5.5及更早版本中的事件冒泡会跳过元素（从跳到document），IE9/Firefox/Chrome/Safari将事件一直冒泡到window 对象;

2.事件捕获

事件捕获：不太具体的节点应更早接收到事件，而最具体的节点应最后接收到事件，目的在于在事件到达预定目标前捕获它；
IE9/Safari/Chrome/Opera/Firefox也都支持事件捕获流模型；尽管“DOM2级事件”规范要求事件应从document对象开始传播，但浏览器都是从window对象开始捕获事件的；

3.DOM事件流

“DOM2级事件”规定的事件流包括三个阶段：事件捕获阶段、处于目标阶段和事件冒泡阶段；

事件处理程序

事件是用户或浏览器自身执行的某种动作，而响应某个事件的函数就是事件处理程序（或事件侦听器）；

1.HTML事件处理程序

某元素支持的每种事件，都有与相应事件处理程序同名的HTML特性，特性值应是能够执行的JavaScript代码：

  • 不能Javascript代码中使用未经转义的HTML语法字符，如和号（&）、双引号（””）、小于号（<）或大于号（>）；
  • HTML中定义的事件处理程序可以包含要执行的具体动作，也可以调用在页面其他地方定义的脚本；

2.DOM0级事件处理程序

  • 每个元素（包括window和document）都有事件处理程序属性，将属性值设置为一个函数即可指定事件处理程序；
  • DOM0级方法指定的事件处理程序是在元素的作用域中运行，即程序中的this引用当前元素；
  • 也可删除通过DOM0级方法指定的事件处理程序：btn.onclick = null;；

3.DOM2级事件处理程序

“DOM2级事件”定义了addEventListener()和removeEventListener()方法，用于处理指定和删除事件处理程序的操作，接受3个参数：要处
理的事件名、作为事件处理程序的函数和布尔值；若布尔值为true，表示在捕获阶段调用事件处理程序，若为false，表示在冒泡阶段调用事件处理程序；

  • DOM2级添加的事件处理程序也是在其依附的元素的作用域中运行；
  • DOM2级方法可以添加多个事件处理程序；
  • 通过addEventListener()添加的事件处理程序只能使用removeEventListener()移除，且移除时传入的参数与添加时相同，通过addEventListener()添加的匿名函数将无法移除；
  • 一般将事件处理程序添加到事件流的冒泡阶段，可以最大限度地兼容，最好只在需要在事件到达目标之前截获它的时候将其添加到捕获阶段；

4.IE事件处理程序

IE中有类似方法attachEvent()和detachEvent()，接受相同的两个参数：事件处理程序名称与事件处理程序函数；

  • 使用attachEvent()方法的事件处理程序在全局作用域中运行，this等于window；
  • attachEvent()方法也可为一个元素添加多个事件处理程序，但以相反的顺序被触发；
  • attachEvent()添加的事件可通过detachEvent()移除，但必须提供相同的参数，添加的匿名函数将不能被移除；

5.跨浏览器的事件处理程序

addHandler()视情况分别使用DOM0级方法、DOM2级方法或IE方法来添加事件，接受3个参数：要操作的元素、事件名称和事件处理程序函数；
addHandler()对应的方法removeHandler()接受相同的参数，目的是移除之前添加的事件处理程序，若其他方法无效，默认采用DOM0级方法；

事件对象

在触发DOM上的某个事件时，会产生事件对象event，对象中包含着所有与事件有关的信息：导致事件的元素、事件的类型以及其他与特定事件相关的信息；

1.DOM中的事件对象

兼容DOM的浏览器会将event对象传入到事件处理程序中；在通过HTML特性指定事件处理程序时，变量event中保存着event对象；event对象包含与创建它的特定事件有关的属性和方法：

属性/方法 类型 说明
bubbles Boolean 表明事件是否冒泡
cancelable Boolean 表明是否可以取消事件的默认行为
currentTarget Element 其事件处理程序当前正在处理事件的元素
defaultPrevented Boolean 为true 表示已经调用了preventDefault()
detail Integer 与事件相关的细节信息
eventPhase Integer 调用事件处理程序的阶段
preventDefault() Function 取消事件的默认行为，cancelable为true则可使用该方法
stopImmediatePropagation() Function 取消事件进一步捕获或冒泡，同时阻止任何事件处理程序被调用
stopPropagation() Function 取消事件进一步捕获或冒泡，bubbles为true则可使用该方法
target Element 事件的目标
trusted Boolean true表示事件是浏览器生成，false表示事件由开发人员创建
type String 被触发的事件的类型
view AbstractView 与事件关联的抽象视图，等同于发生事件的window对象

事件处理程序内部，对象this始终等于currentTarget的值，而target只包含事件的实际目标；若直接将事件处理程序指定给目标元素，则this、currentTarget 和target包含值相同；

2.IE中的事件对象

  • 使用DOM0级方法添加事件处理程序时，event对象作为window对象的属性存在；
  • 若事件处理程序使用attachEvent()添加，event对象会作为参数被传入事件处理程序函数中；
  • 若通过HTML特性指定事件处理程序，可通过名为event的变量访问event对象；

IE的所有事件对象都包含下列属性和方法：

属性/方法 类型 说明
cancelBubble Boolean 默认值为false，设置为true可以取消事件冒泡
returnValue Boolean 默认值为true，设置为false可以取消事件的默认行为
srcElement Element 事件的目标
type String 被触发的事件的类型

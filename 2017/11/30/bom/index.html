window对象

BOM的核心对象是window，表示浏览器的一个实例；在浏览器中，它既是通过JavaScript访问浏览器窗口的接口，又是ECMAScript规定的Global对象；在网页中定义的任何一个对象、变量和函数，都以window作为其Global对象；

1.全局作用域

  • 所有在全局作用域中声明的变量、函数都会变成window对象的属性和方法；
  • 使用var语句添加的window属性（即全局变量）的[[Configurable]]特性值为false，所以全局变量不能通过delete操作符删除，而直接在window对象上的定义的属性可以；
  • 尝试访问未声明的变量会抛出错误，但通过查询window对象，可以知道某个可能未声明的变量是否存在；

注：Windows Mobile平台的IE浏览器不允许通过window.property = value的形式直接在window对象上创建新的属性或方法，但在全局作用域中声明的所有变量和函数会变成window对象的成员；

2.窗口关系及框架

若页面中包含框架，则每个框架都有自己的window对象，并且保存在frames集合中；在frames集合中，可以通过数值索引（从0 开始，从左至右，从上到下）或框架名称来访问相应window对象；每个window对象都有name属性，其中包含框架的名称；
top对象始终指向最外层框架，即浏览器窗口，使用它可以确保在一个框架中正确访问另一个框架；parent对象始终指向当前框架的直接上层框架，在没有框架时parent等于top；self对象始终指向window，self和window对象可以互换使用；

注：

  • 除非最高层窗口是通过window.open()打开的，否则其window对象的name属性不包含任何值；
  • 在使用框架时，在每个框架中定义的全局变量会成为框架中window对象的属性；
  • 由于每个window对象都包含原生类型的构造函数，因此每个框架都有一套构造函数，这些构造函数一一对应，但并不相等；

3.窗口位置

IE/Safari/Opera/Chrome提供了screenLeft和screenTop属性，用于表示窗口相对于屏幕左边和上边的位置，Firefox/Safari/Chrome在screenX和screenY属性中提供相同的窗口位置信息；跨浏览器支持：

moveTo()和moveBy()可以将窗口精确地移动到一个新位置；window.moveTo()接收新位置的x和y坐标值，window.moveBy()接收在水平和垂直方向上移动的像素数；

4.窗口大小

resizeTo()和resizeBy()方法可以调整浏览器窗口的大小；window.resizeTo()接收浏览器窗口的新宽度和新高度，window.resizeBy()接收新窗口与原窗口的宽度和高度之差；

5.导航和打开窗口

window.open()方法既能导航到特定的URL，也可打开新的浏览器窗口，接收4个参数：要加载的URL、窗口目标、特性字符串以及是否取代历史记录中当前加载页面的布尔值；通常只需第一个参数，最后的参数只在不打开新窗口的情况下使用；

弹出窗口

如果window.open()第二个参数不是已经存在的窗口或框架，会根据第三个参数的字符串创建新窗口或新标签页；未传入第三个参数则打开一个带有全部默认设置的新浏览器窗口；在不打开新窗口的情况下，忽略第三个参数；
window.open()方法返回一个指向新窗口的引用；close()可以关闭新打开的窗口；新创建的window对象有opener属性保存着打开它的原始窗口对象，该属性只在弹出窗口中的最外层window对象（top）中有定义，而且指向调用window.open()的窗口或框架；

弹出窗口屏蔽程序

若是浏览器内置屏蔽程序阻止的弹出窗口，则window.open()很可能返回null；若是浏览器扩展或其他程序阻止的弹出窗口，window.open()通常抛出一个错误；

6.间歇调用和超时调用

超时调用需要使用window.setTimeout()方法，接受两个参数：要执行的代码（可以是包含JavaScript代码的字符串，也可以是函数）和以毫秒表示的时间；该方法会返回数值ID，超时调用ID是计划执行代码的唯一标识符，可以用来取消超时调用（调用clearTimeout()）；
间歇调用的方法是setInterval()，接受参数与setTimeout()相同，同样也返回间歇调用ID，clearInterval()方法可取消未执行的间歇调用；

7.系统对话框

  • alert()方法接受字符串并将其显示给用户；
  • confirm()方法返回布尔值：true表示单击了OK，false 表示单击了Cancel或单击了关闭按钮；
  • prompt()方法用于提示用户输入文本，接受两个参数：显示给用户的文本提示和文本输入域的默认值，单击OK按钮返回文本输入域的值，单击了Cancel或关闭对话框则返回null；

location对象

location对象提供了与当前窗口中加载的文档有关的信息和一些导航功能；location对象既是window对象的属性，也是document对象的属性，即window.location和document.location引用的是同一对象；

属性名 举例 说明
hash “#contents” 返回URL中的hash（#号后跟零或多个字符），若不含散列，则返回空字符串
host “www.wrox.com:80" 返回服务器名称和端口号（如果有）
hostname “www.wrox.com" 返回不带端口号的服务器名称
href “http:/www.wrox.com" 返回当前加载页面的完整URL。也可调用location对象的toString()方法
pathname “/WileyCDA/“ 返回URL中的目录和（或）文件名
port “8080” 返回URL中指定的端口号，若不含端口号，则返回空字符串
protocol “http:” 返回页面使用的协议。通常是http:或https:
search “?q=javascript” 返回URL的查询字符串，该字符串以问号开头
  • location.assign()方法会立即打开传入的新URL并在浏览器历史记录中生成一条记录；location.href或window.location设置为一个URL值，会以该值调用assign()方法，修改location其他属性（hash/search/hostname/pathname/port）也可改变当前加载的页面；
  • replace()方法接受参数：要导航到的URL，会导致浏览器位置改变，但不会在历史记录中生成新记录；
  • reload()会重新加载当前显示的页面，若要强制从服务器重新加载需要传递参数true；

navigator对象

navigator对象可用于检测显示网页的浏览器类型；

1.检测插件

非IE浏览器可以使用plugins数组来检测浏览器是否安装了特定插件，数组中的每一项包含下列属性：

  • name–插件的名字；
  • description–插件的描述；
  • filename–插件的文件名；
  • length–插件所处理的MIME 类型数量；

2.注册处理程序

  • registerContentHandler()方法接收三个参数：要处理的MIME 类型、可以处理该MIME类型的页面的URL以及应用程序的名称；
  • registerProtocolHandler()方法接收三个参数：要处理的协议、处理该协议的页面的URL 和应用程序的名称；

screen对象和history对象

1.screen对象

screen对象只用来表明客户端的能力，其中包括浏览器窗口外部的显示器信息；

2.history对象

  • go()方法可以在用户的历史记录中任意跳转，接受一个参数：表示跳转页面数的整数值（负数表示向后跳转）；若传递一个字符串参数，浏览器会跳转到历史记录中包含该字符串的第一个位置；
  • back()和forward()可以模仿浏览器的“后退”和“前进”按钮；
  • length属性保存着历史记录的数量；

面向对象的程序设计

发表于 2018-01-19 | 分类于 《JavaScript高级程序设计》读书笔记

1 理解对象

1.1 属性类型

数据属性

数据属性包含一个数据值的位置，可以读取和写入值：

  • [[Configurable]]（默认为true）：表示能否通过delete删除属性从而重新定义属性，能否修改属性的特性，或者能否把属性修改为访问器属性；
  • [[Enumerable]]（默认为true）：表示能否通过for-in循环返回属性；
  • [[Writable]]（默认为true）：表示能否修改属性的值；
  • [[Value]]（默认为undefined）：包含该属性的数据值；

ECMAScript 5的Object.defineProperty()方法可以修改属性默认的特性，接收三个参数：属性所在对象、属性名和一个描述符对象（描述符对象的属性必须是：configurable、enumerable、writable和value，可以修改对应的特性值）；

一旦把属性定义为不可配置的，就不能再把它变回可配置，此时再调用该方法修改除writable之外的特性都会导致错误；

调用Object.defineProperty()方法时，若不指定，configurable、enumerable和writable特性的默认值都是false；

访问器属性

访问器属性：在读取访问器属性时会调用getter函数，返回有效的值；在写入访问器属性会调用setter函数并传入新值；

  • [[Configurable]]（默认为true）：表示能否通过delete删除属性从而重新定义属性，能否修改属性的特性，或者能否把属性修改为数据属性；
  • [[Enumerable]]（默认为true）：表示能否通过for-in循环返回属性；
  • [[Get]]（默认为undefined）：在读取属性时调用的函数；
  • [[Set]]（默认为undefined）：在写入属性时调用的函数；

必须使用Object.defineProperty()定义访问器属性；

不一定非要同时指定getter和setter，只指定getter意味着属性是不能写，只指定setter函数的属性也不能读，而在严格模式下会抛出错误；还有非标准方法__defineGetter__()和__defineSetter__()创建访问器属性；

1.2 定义多个属性

ECMAScript 5中的Object.defineProperties()方法可通过描述符一次定义多个属性，接收两个对象参数：第一个对象是要添加和修改其属性的对象，第二个对象的属性与第一个对象中要添加或修改的属性一一对应；

1.3 读取属性的特性

ECMAScript 5的Object.getOwnPropertyDescriptor()方法可以取得给定属性的描述符，接收两个参数：属性所在的对象和要读取其描述符的属性名称，返回一个对象；若是访问器属性，则对象属性有configurable、enumerable、get和set，若是数据属性，则对象属性有configurable、enumerable、writable和value

2 创建对象

2.1 工厂模式

工厂模式抽象了创建具体对象的过程，但没有解决对象识别的问题；

2.2 构造函数模式

创建自定义的构造函数，从而定义自定义对象类型的属性和方法；

与工厂模式相比，该模式没有显式地创建对象，直接将属性和方法赋给this对象，没有return语句；

创建Person的新实例，必须使用new操作符，调用构造函数会经历以下步骤：创建新对象—>将构造函数的作用域赋给新对象—>执行构造函数中的代码—>返回新对象；对象的constructor属性指向构造函数，可用来标识对象类型；

将构造函数当作函数

构造函数与其他函数的唯一区别，就在于调用的方式不同；任何函数通过new操作符来调用，就可以作为构造函数；任何函数若不通过new操作符来调用，那么就和普通函数一样；

还可以使用call()（或者apply()）在某个特殊对象的作用域中调用Person()函数；（示例是在对象o的作用域中调用的）

构造函数的问题

构造函数的主要问题，就是每个方法都要在每个实例上重新创建一遍，不同对象的同名方法不是同一个Function的实例；可以把函数定义转移到构造函数外部来解决这个问题，此时需要将相关属性设置成等于全局函数；

2.3 原型模式

每个函数都有prototype属性，指向的对象是包含可以由特定类型的所有实例共享的属性和方法，即prototype就是通过调用构造函数而创建的那个对象实例的原型对象；使用原型对象可以使所有对象实例共享其所包含的属性和方法；

理解原型对象

只要创建了新函数，就会根据特定的规则为函数创建一个prototype属性，该属性指向函数的原型对象；默认情况下，所有原型对象都会获得一个constructor属性，该属性包含一个指向prototype属性所在函数的指针；
当调用构造函数创建新实例后，该实例内部将包含一个[[Prototype]]指针，指向构造函数的原型对象（没有标准的方式访问[[Prototype]]，但部分浏览器支持属性__proto__）；[[Prototype]]存在于实例与构造函数的原型对象之间，isPrototypeOf()方法可以确定对象之间是否存在这种关系；
ECMAScript 5新增的Object.getPrototypeOf()会返回[[Prototype]]的值；

ECMAScript 5的Object.getPrototypeOf()方法返回[[Prototype]]的值（对象的原型）；

为对象实例添加属性时，会屏蔽原型对象中保存的同名属性；即使将该属性设置为null，也只会在实例中设置该属性，不会恢复其指向原型的连接；但使用delete操作符可完全删除实例属性，从而能重新访问原型中的属性；hasOwnProperty()方法可以检测一个属性存在于实例中，还是存在于原型中；ECMAScript 5的Object.getOwnPropertyDescriptor()方法只用于实例属性，要取得原型属性的描述符，须在原型对象上调用Object.getOwnProperty-Descriptor()方法；

原型与in操作符

in操作符会在通过对象能访问给定属性时返回true（无论该属性存在于实例中还是原型中）；同时使用hasOwnProperty()方法和in操作符可以确定该属性存在于对象中还是原型中；

for-in循环返回所有能通过对象访问的、可枚举的属性，其中既包括存在于实例中的属性，也包括存在于原型中的属性，屏蔽了原型中不可枚举属性的实例属性也会在for-in循环中返回；Object.keys()方法可取得对象上所有可枚举的实例属性，接收对象作为参数，返回包含所有可枚举属性的字符串数组；Object.getOwnPropertyNames()方法可取得所有实例属性（无论否可枚举）；

原生对象的原型

所有原生引用类型都在其构造函数的原型上定义了方法；通过原生对象的原型，不仅可以取得所有默认方法的引用，而且也可以定义新方法；

2.4 组合使用构造函数模式和原型模式

构造函数模式用于定义实例属性，原型模式用于定义方法和共享的属性；

2.5 动态原型模式

动态原型模式把所有信息都封装在构造函数中，而通过在构造函数中初始化原型，又保持了同时使用构造函数和原型的优点；

2.6 寄生构造函数模式

创建函数的作用仅仅是封装创建对象的代码，然后再返回新创建的对象；返回的对象与构造函数或与构造函数的原型属性之间没有关系,不能依赖instanceof操作符来确定对象类型;

2.7 稳妥构造函数模式

稳妥对象是指没有公共属性且其方法也不引用this的对象；稳妥构造函数遵循与寄生构造函数类似的模式，但新创建对象的实例方法不引用this，而且不使用new操作符调用构造函数；

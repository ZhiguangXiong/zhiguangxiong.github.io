1.单向数据流

父子组件之间传递数据是单向下行绑定的，意味着父级数据的更新会通过 prop 向下流动到子组件中，子组件中的值将会是最新的。但是反过来不行。 这样是为了防止子组件意外改变父级组件的状态，从而导致应用的数据流向难以理解，这意味着你不应该在一个子组件内部 直接改变 通过 prop 传进来的数据。 如果你这样做了，Vue 会在浏览器的控制台中发出警告。 有两种情况需要改变一个 props 的值：

组件

发表于 2018-04-11 | 分类于 Vue.js学习笔记

组件是可复用的 Vue 实例，因此与 Vue 实例具有相同的选项，如 data、methods、computed、watch、components 等， 不同的是组件还具有 props 属性，父组件可通过该属性给子组件传递数据。此外，组件还具有一系列钩子函数可以使用。

1.data 必须是一个函数

与 Vue 根实例不同的是，组件中的 data 必须是一个函数，同时返回一个对象，这样每个组件就可以单独的维护各自的数据， 否则组件的数据会相互影响。具体实例可参照官网。

模板语法

发表于 2018-03-05 | 分类于 Vue.js学习笔记

组件

发表于 2018-04-11 | 分类于 Vue.js学习笔记

组件是可复用的 Vue 实例，因此与 Vue 实例具有相同的选项，如 data、methods、computed、watch、components 等， 不同的是组件还具有 props 属性，父组件可通过该属性给子组件传递数据。此外，组件还具有一系列钩子函数可以使用。

1.data 必须是一个函数

与 Vue 根实例不同的是，组件中的 data 必须是一个函数，同时返回一个对象，这样每个组件就可以单独的维护各自的数据， 否则组件的数据会相互影响。具体实例可参照官网。

2.组件注册

组件可通过两种方式进行注册：全局注册与局部注册。全局注册的组件必须在生成根实例之前进行注册， 注册成功后，在任何组件中都可以使用。局部组件必须使用 ES6 module 语法引入进来，放入至 components 配置中。 组件全局注册需要注意的是，如果你将所有的组件全部进行了全局注册，那么当你使用了类似于 webpack 这样的构建系统时， 如果你不再使用这个全局注册的组件了，但是这个组件还会打包在你的构建结果中，这会造成 JavaScript 下载量无谓的增加。

3.组件命名

组件名可以为 camelCase 驼峰形式，也可以为 kebab-case 短横线分隔形式，需要注意的是无论以何种方式进行注册的， 在 DOM 中直接使用时，都需要使用 kebab-case 形式，否则无效。 在字符串模板中或者单文件组件中两种形式皆可。

4.单个根元素

组件的模板中只能有一个根元素， 组件包含多个同级的根元素时，Vue 会给予警告： every component must have a single root element，你可以将模板的内容包裹在一个父元素内，来修复这个问题。

5.数据的传递

父组件通过属性的绑定向子组件传递数据，而子组件通过 props 配置来接受父组件传递的数据，接收到的数据可作为一个属性在子组件内被访问到。 子组件通过 $emit 向父组件发送消息，父组件需要在子组件上监听这个事件，同时绑定一个回调函数，在子组件 emit 了相应事件后执行相应回调。 同时子组件可以将所要传递 给父组件的数据作为第二个参数传递给父组件内的回调函数。下面是一个 todoList 的例子：
